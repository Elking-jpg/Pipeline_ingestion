
"""
===============================================================================
COMPILADO TÉCNICO: ARQUITECTURA SÍNCRONA VS. ASÍNCRONA EN CONSUMO DE APIs
===============================================================================

1. PARADIGMA SÍNCRONO (Secuencial / Bloqueante) - Ejemplo: 'requests'
-------------------------------------------------------------------------------
* DEFINICIÓN: El hilo de ejecución procesa las instrucciones en un orden lineal
  estricto. Cuando realiza una petición de red (I/O Bound), el programa se
  "congela" (bloquea) por completo. No ejecuta ninguna otra línea hasta que los
  paquetes de datos viajan por internet, el servidor responde y la descarga local 
  finaliza.
* ANATOMÍA DEL TIEMPO: El tiempo total de ejecución es acumulativo. 
  Fórmula: T_total = T_peticion1 + T_peticion2 + ... + T_peticionN.
* GESTIÓN DE ERRORES: Requiere envolver el bloque interno en un 'try-except'
  para capturar excepciones de red (Timeouts, ConnectionErrors) y usar un
  'continue' en el bucle para evitar el colapso secuencial del pipeline.

2. PARADIGMA ASÍNCRONO (Concurrente / No Bloqueante) - Ejemplo: 'asyncio' + 'aiohttp'
-------------------------------------------------------------------------------
* DEFINICIÓN: Cambia la naturaleza de las funciones convirtiéndolas en Corrutinas 
  ('async def'). El control de la ejecución pasa a estar coordinado por un motor 
  interno llamado Event Loop (Bucle de Eventos). Cuando una corrutina inicia una 
  operación de red, libera voluntariamente el procesador mediante la palabra 
  clave 'await'. Mientras la red procesa la solicitud, el Event Loop avanza con 
  las demás tareas en segundo plano.
* ANATOMÍA DEL TIEMPO: Las operaciones ocurren en simultáneo (concurrencia). 
  El tiempo total de ejecución es óptimo.
  Fórmula: T_total = Máximo(T_peticion_individual) + micro-delay de ordenamiento.
* COMPONENTES CLAVE:
    - 'aiohttp.ClientSession()': Abre un único pool de conexiones persistentes 
      para reutilizar el canal de red sin el costo de abrir sockets individuales.
    - 'asyncio.gather(*tasks)': Desempaquetador y coordinador masivo de las promesas 
      de ejecución pendientes.
* GESTIÓN DE ERRORES: Los errores se mitigan aislando el 'try-except' dentro de 
  cada corrutina atómica, retornando un objeto 'None' como bandera de fallo, el cual 
  se depura mediante filtros lógicos en el paso final de consolidación de datos.
===============================================================================
"""
